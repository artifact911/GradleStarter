Создадим директорию buildSrc и внетри файл build.gradle. Обновим проект. Наша новая
    дирктория стала модулем, т.к. это зарезервированная директория для всех наших самописных
    тасок, скриптов и т.д. Любой цикл, который мы запустим, сначала запустит в работу
    модуль buildSrc.

    По умолчанию buildSrc уже реализовывает и java и groovy plugins одновременно.
    Добавим плагин:
        plugins {
            id 'groovy'
        }
    и дефолтные директории (main, java, resources etc)

    Создав groovy plugin в соответствующей директоии мы можем его импортировать:
        apply plugin: CustomPlugin
    Но так у нас появляетс импорт. Хоти избавиться, тогда:
        apply plugin: com.art.plugin.CustomPlugin
    Но есть другой способ: создаем в пакете ресурсов пакеты META-INF/gradle-plugins.
    Это зарезервированные пакеты. Внутри создаем
            custom-art.properties
    где custom-art это будет ID нашего плагина. А внутри файла одно единственное поле:
            implementation-class=com.art.plugin.CustomPlugin

    Теперь наш плагин мы можем добавлять через плагины
            plugins {
                id 'custom-art'
            }
